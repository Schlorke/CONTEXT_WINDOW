# Compliance Checklist for AI Interfaces

## WCAG 2.2 AA (Web Content Accessibility Guidelines)

### Streaming Responses

Use ARIA live regions to announce text as it arrives:

```jsx
<div role="log" aria-live="polite" aria-label="AI response">
  {message.content}
</div>
```

Every new chunk of text is announced to screen readers as it arrives.

### Keyboard Navigation

- Tab through messages.
- Regenerate button always keyboard-accessible.
- Ctrl+Enter to send (common chat app pattern).

### Focus Management

Shift focus to response after completion for screen reader announcement:

```jsx
const responseRef = useRef(null);
useEffect(() => {
  if (newResponse) {
    responseRef.current?.focus();
  }
}, [newResponse]);
```

### Color Contrast

- All UI elements: 4.5:1 contrast minimum.
- Test with tools: WebAIM, Axe DevTools.

### Screen Reader-Friendly Rendering

Good:

```jsx
<div>
  <span aria-hidden="true">🤖</span>
  <span>AI Assistant</span>
</div>
```

Bad (nonsense to screen readers):

```jsx
<div>🤖 AI</div>
```

## ISO 42001 (AI Management)

### Documentation Requirements

- Document all model versions, prompt changes, usage patterns.
- Maintain audit trail of user interactions.
- Risk assessment for high-stakes use cases.

### Audit Log Schema

```prisma
model AuditLog {
  id String @id
  userId String
  action String  // "SENT_MESSAGE", "GENERATED_RESPONSE", etc.
  modelVersion String  // Claude 3.5 Sonnet 20241022
  systemPromptVersion String  // v2.3
  createdAt DateTime
}
```

## EU AI Act (if serving EU users)

### Transparency Obligation

- Clearly state: "This is AI, not human."
- Always label responses as generated by AI.

### High-Risk System Registration

- If applicable to your SaaS (e.g., high-stakes decisions), register system.
- Document risk assessment and mitigation measures.

### Data Retention Policies

- Clearly state how long user data is retained.
- Provide data deletion mechanisms.
- Document GDPR compliance (if applicable).
